# セキュリティ運用手順書

## ログイン問題のトラブルシューティング

### ログインできない場合の対処法

ログインできない場合は、以下の手順を確認してください。

1. **パスワードの確認**
   - パスワードが正しく入力されているか確認
   - 大文字・小文字、数字、記号が正しく入力されているか確認
   - Caps Lockが有効になっていないか確認

2. **アカウントロックの確認**
   - 連続してログインに失敗した場合、アカウントがロックされる可能性があります
   - ロック解除には管理者への連絡が必要です

3. **ネットワーク接続の確認**
   - ネットワーク接続が正常か確認
   - VPN接続が必要な場合は、VPNが確立されているか確認

4. **ブラウザのキャッシュクリア**
   - ブラウザのキャッシュをクリアしてから再度ログインを試行

### パスワードリセット手順

パスワードをリセットする場合は、以下の手順に従ってください。

1. パスワードリセットページにアクセス
2. ユーザーIDとメールアドレスを入力
3. メールに送信されたリセットリンクをクリック
4. 新しいパスワードを設定（8文字以上、大文字・小文字・数字・記号を含む）

## インシデント対応手順

### インシデント発生時の初動対応

インシデントが発生した場合、以下の手順に従って対応してください。

1. **影響範囲の確認**
   - どのシステムが影響を受けているか確認
   - 影響を受けているユーザー数を確認
   - データの漏洩や改ざんの有無を確認

2. **緊急度の判定**
   - 緊急度を判定（高/中/低）
   - 高緊急度の場合は、即座にエスカレーション

3. **証跡の保全**
   - ログファイルのバックアップ
   - スクリーンショットの取得
   - 関連するファイルの保存

4. **報告**
   - インシデント管理チームに報告
   - 報告書の作成

### ログファイルの確認方法

ログファイルを確認する場合は、以下の手順に従ってください。

1. ログ管理システムにログイン
2. 対象期間を指定して検索
3. 該当するログエントリを確認
4. 必要に応じてログをエクスポート

**注意**: ログファイルは証跡として重要です。削除や改ざんは絶対に行わないでください。

## セキュリティ設定

### ファイアウォール設定

ファイアウォールの設定を変更する場合は、以下の手順に従ってください。

1. ファイアウォール管理画面にアクセス
2. ルールを追加または変更
3. 変更内容を確認
4. 設定を適用

**警告**: ファイアウォール設定の誤りは、システム全体に影響を与える可能性があります。変更前には必ず承認を得てください。

### アクセス制御

アクセス制御を設定する場合は、以下の原則に従ってください。

- 最小権限の原則: 必要最小限の権限のみを付与
- 定期的な権限レビュー: 四半期ごとに権限をレビュー
- 不要な権限の削除: 不要になった権限は即座に削除

## バックアップと復旧

### バックアップの取得

バックアップを取得する場合は、以下の手順に従ってください。

1. バックアップ対象を選択
2. バックアップスケジュールを設定
3. バックアップの実行
4. バックアップの検証

### データ復旧手順

データを復旧する場合は、以下の手順に従ってください。

1. 復旧対象のデータを特定
2. 適切なバックアップを選択
3. 復旧先を指定
4. 復旧を実行
5. 復旧データの検証

**重要**: データ復旧は慎重に行ってください。誤った復旧は、現在のデータを上書きする可能性があります。
